# FOOTER_AND_SITEMAP_REGULATION.md

## Главный принцип

Все новые страницы сайта Make All Perfect должны добавляться через единый реестр:

```text
src/content/navigation/siteMap.ts
```

Этот файл используется для:
- Footer;
- sitemap.xml;
- будущей навигации;
- legal-ссылок;
- контроля структуры сайта.

## Как добавить новую страницу

```ts
{ 
  title: "Название страницы",
  href: "/page-url",
  footer: true,
  sitemap: true,
  priority: 0.7,
  changeFrequency: "monthly"
}
```

## Правила

1. Если страница должна быть в подвале — `footer: true`.
2. Если страница должна быть в sitemap.xml — `sitemap: true`.
3. Если страница техническая и не должна индексироваться — `sitemap: false`.
4. Если страница не должна отображаться в подвале — `footer: false`.
5. Не прописывать ссылки вручную внутри Footer.
6. Footer должен брать данные только из `getFooterGroups()`.
7. sitemap.ts должен брать данные только из `getSitemapItems()`.

## Термины в футере

В футере используем формулировку:

```text
Юридическое лицо
```

Не используем слово `Оператор` в видимом футере, потому что оно юридически корректно для политики персональных данных, но в подвале сайта выглядит слишком канцелярски.

## Подключение Footer

В основном layout проекта добавить:

```tsx
import Footer from "@/components/layout/Footer";
```

И вставить:

```tsx
<Footer />
```
