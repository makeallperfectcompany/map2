# CONTENT TAXONOMY REGULATION

Единая таксономия контента для фильтров, поиска, тегирования и SEO на сайте Make All Perfect.

---

## 1. Источник правды

Единственный источник всех категорий, отраслей, услуг и тегов:

```
src/content/taxonomy.ts
```

Все изменения таксономии — только через этот файл.

---

## 2. Обязательные поля для статьи

Каждая статья в `src/content/articles/` должна содержать:

```ts
contentType: "article";        // строго
category: string;              // из contentCategories
tags: string[];                // из tags (минимум 1)
service: string | null;        // из services или null
industry: string | null;       // из industries или null
priority: number;              // 0–200, выше = важнее
```

---

## 3. Обязательные поля для кейса

Каждый кейс в `src/content/cases/` должен содержать:

```ts
contentType: "case";           // строго
category: string;              // из caseCategories
tags: string[];                // из tags (минимум 1)
service: string | null;        // из services или null
industry: string | null;       // из industries или null
priority: number;              // 0–200, выше = важнее
date: string;                  // дата публикации
```

## 5. Как добавлять новые значения

1. Открыть `src/content/taxonomy.ts`.
2. Добавить значение в соответствующий массив (contentCategories / caseCategories / industries / services / tags).
3. Использовать новое значение в content-файле статьи или кейса.
4. В отчёте указать:
   - какие новые category / service / industry / tag были добавлены;
   - почему не подошли существующие значения.

**Запрещено:**
- придумывать дубли (например, "WB" вместо "Wildberries", "Внешняя реклама" вместо "Внешняя реклама для Wildberries");
- использовать значения, которых нет в taxonomy.ts, без предварительного добавления;
- изменять написание существующего значения.

## 6. Фильтрация

### /blog (BlogIndexSection)
Категории собираются автоматически из всех статей:
```ts
const categories = ["Все", ...Array.from(new Set(articles.map((a) => a.category)))];
```

### /cases (CasesIndexSection)
Категории собираются автоматически из всех кейсов:
```ts
const filters = ["Все", ...Array.from(new Set(cases.map((c) => c.category)))];
```

Не дублировать категории вручную.

---

## 7. Будущее: поиск

При подключении поиска используйте:
- `tags` — для фильтрации по тематике;
- `category` — для фильтрации по рубрикам;
- `service` — для показа связанных услуг;
- `industry` — для фильтрации по отраслям;
- `contentType` — для разделения типов контента в результатах.

## 8. Кем поддерживается

Регламент поддерживается OpenClaw при создании нового контента.
При конфликте данных — приоритет у `src/content/taxonomy.ts`.
